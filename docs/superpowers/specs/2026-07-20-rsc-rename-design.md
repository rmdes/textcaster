# RSC rename — design

**Status:** approved design, pre-plan
**Date:** 2026-07-20

Textcaster is renamed to **RSC — Really Simple Conversations**. The name is
an homage to RSS's acronym pattern ("Really Simple Syndication") and names
the app's differentiator: conversations that travel as RSS.

## Identity decisions (settled with user)

- **Name:** RSC. Long form **Really Simple Conversations**, used as tagline
  in README, About page, and repo description.
- **GitHub repo:** renamed to `github.com/rmdes/rsc` (GitHub redirects the
  old URL; local `origin` re-pointed).
- **Docker image:** `rmdes/rsc` for future builds.
- **Env vars:** all 16 `TEXTCASTER_*` → `RSC_*`, clean break, **no fallback
  shim** (we control every deployment).
- **Domains:** unchanged for now — `textcaster.app` + `alice.`/`bob.` keep
  serving; name is deliberately untied from domain. Domain migration (e.g.
  textca.st is owned but unused) is a separate future effort.
- **Local checkout:** stays `~/textcaster`. Purely local, rename any time.

## Rename surfaces

~141 files mention the name; they cluster into:

1. **Env vars** — `core/src/config.ts` and every reader; `compose.yaml`,
   `compose.prod.yaml`, root + core `.env.example`, `scripts/generate-env.sh`,
   `Caddyfile`, `cloudron/start.sh` (incl. its `tc_` helper prefix,
   cosmetic). Compose `container_name: textcaster-*` (7 across both files)
   → `rsc-*`; dev-only, containers recreate harmlessly.
2. **Packages** — root/core/web `package.json` names + `package-lock.json`
   (regenerated by `npm install`).
3. **User-facing strings** —
   - web page titles, headers, About page, login/register/reset pages;
   - auth email subjects in `core/src/auth.ts` ("Your Textcaster login
     link", etc.);
   - mail-from default `textcaster@${mailHost}` (`core/src/config.ts:81`);
   - outbound fetch user-agent `core/src/domain/ingest.ts:35`
     (`Textcaster/0.1 (+…rmdes/textcaster)` → `RSC/0.1 (+…rmdes/rsc)`);
   - feed-output fallback host `textcaster.invalid` → `rsc.invalid`
     (`core/src/domain/feed.ts:53,124,138,205`);
   - startup log line (`core/src/server.ts:84`);
   - localStorage draft prefix `textcaster:draft:` → `rsc:draft:`
     (`web/src/lib/draft.ts:5`) — in-flight drafts are silently discarded;
     accepted;
   - `LICENSE` line 3 ("Textcaster contributors" → "RSC contributors").
4. **Tests** — core: `auth.test.ts:42` asserts the
   `textcaster.session_token` cookie name and `ingest.test.ts:29` the
   user-agent (both fail outright post-rename); `config.test.ts`,
   `config-admin.test.ts`, `push.test.ts`, `push-in.test.ts`,
   `federation-live.test.ts` are dense with `TEXTCASTER_*` keys. Web: ~30
   `textcaster.session_token` literals across `auth.actions.test.ts`,
   `page.actions.test.ts`, `accounts.server.test.ts`,
   `edit.actions.test.ts`, `session.test.ts`, plus `draft.test.ts` keys.
   (Web prod code never reads the prefixed cookie name — matching is
   substring-only, `session.ts:7` — so the prefix change can't break the
   proxy.)
5. **Cloudron package** — manifest `title`, `description`, `tagline`,
   `website`, `tags` (the `"textcasting"` tag is removed outright; see
   invariant 1); `nginx.conf:4` pid path (tmpfs runtime path, safe to
   rename); `cloudron/README.md` env references.
6. **Docs & design system** — `README.md`, `CLAUDE.md`,
   `docs/superpowers/documentation/RUNNING.md`, `ideas.md` live references;
   `design-system/textcaster/` → `design-system/rsc/` (git mv, plus every
   live reference to that path). Live repo links (About page, layout footer)
   are rewritten to `rmdes/rsc` even though GitHub redirects.
7. **Historical docs are NOT rewritten** — specs/plans/reviews under
   `docs/superpowers/` keep their "Textcaster" text per house convention
   (executed docs are records). Links from live pages to historical doc
   *filenames* (e.g. About → `2026-07-15-textcaster-design.md`) keep the
   old filename. This spec documents the rename; history stays as written.

## Attribution (sidebar + About page)

Credit to Dave Winer is a promise made and kept — in README, the sidebar
About blurb, and the About page. The rename changes *framing*, never
*credit*:

- **Credit line** (sidebar, About, README): "Inspired by Dave Winer's
  [Textcasting](https://textcasting.org) and
  [rss.chat](https://github.com/scripting/rss.chat)." — "inspired by"
  replaces "built on" (credits his work without claiming it as our
  platform).
- **Sidebar blurb** (`web/src/routes/+page.svelte`): existing paragraph with
  "RSC — Really Simple Conversations —" as the subject, then the credit
  line.
- **About page** (`web/src/routes/about/+page.svelte`): keep structure and
  the full credits list ("on whose shoulders") untouched in substance;
  rewrite the lede to drop the "unifies Dave Winer's lineage / refuses the
  purity lines" origin-story framing; delete "The name is itself an
  attribution."; soften "the rss.chat idea this reimagines" to plain
  interop language. Credit stays; speaking *for* his lineage goes.

## Data-safety invariants

These three are where a naive find-replace destroys live state:

1. **Cloudron manifest `id: "net.textcaster.app"` NEVER changes.** Cloudron
   identifies the installed app by this id; changing it makes the build a
   different app and orphans all three live instances. Only human-facing
   manifest fields change.
2. **Stored-state paths keep their exact values.** `start.sh` keeps
   `RSC_DB="/app/data/textcaster.db"` (live databases stay where they are)
   and the generate-once secret files stay at `/app/data/config/auth_secret`
   / `ops_token` / `admin_email`. Var *names* change; *values* pointing at
   persisted files do not. Local dev default in `config.ts` may become
   `./data/rsc.db` (dev data is disposable).
3. **`cookiePrefix: 'textcaster'` → `'rsc'` logs everyone out once.**
   Accepted and documented: all live sessions invalidate on deploy; users
   re-log-in. No migration shim.

## Out of scope

- Domain migration (textcaster.app → anything else), including textca.st.
- Renaming the local checkout directory.
- Renaming live DB files on Cloudron instances.
- Any feature work; this is a pure rename.

## Verification

- Full suites green: core vitest, web vitest, `tsc`, `svelte-check`.
- `git grep -i textcaster` (tracked files only; run after `npm install` has
  regenerated `package-lock.json`) returns ONLY the allowlist:
  `docs/superpowers/` historical files and their filenames referenced from
  live pages, domain values that legitimately remain (compose/Caddyfile
  `textcaster.app` hosts, `start.sh` DB path), and this spec. "Textcasting"
  as the *interop-convention* term (internal identifiers/comments:
  `listTextcastingPeers`, `source:markdown` marker notes) is NOT renamed —
  it names the protocol we interop with, like "RSS"; the only user-facing
  uses are the credit line and About-page credits, which stay by design.
- Sanitizer twins untouched (`core/src/domain/markdown.ts` /
  `web/src/lib/server/render.ts`) — rename must not brush the XSS gate.

## Rollout

1. Land the rename on `main`; rename the GitHub repo; re-point `origin`.
2. Build image `rmdes/rsc:<tag>` (CloudronManifest + logo symlink dance as
   usual).
3. Staged deploy: update **bob** first → verify data survived, feeds serve,
   login works (fresh session expected per invariant 3) → then alice → then
   textcaster.app.
